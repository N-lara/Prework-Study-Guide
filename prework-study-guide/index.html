HTML

  • the head element holds webpage info for browsers
  • the body element holds info visible to users
  • in the body there is a header main and footer theese are semantic elements to help people read the code and stay orginized
  • needs to be started with !DOCTYPE html
  • ul is unordered list (bullets) ol ia ordered list (numbers) li is list items
  • h2 through h6 is headres 1 being biggest and 6 being smallest only 1 h1 per page
  • title is what is written in tab area on browser
  • use: link rel="stylesheet" href="path to css file" to connect an external style sheet to a html doc
  • html needs an opening and closing tag

CSS

  • .class-name{css} attributes css to only HTML marked with the class name
  • box model says from inside to outside it goes: content > padding> border> margin
  • can be inline (directly in html tag), internal (in head of html) or external (in a .css file)
  • css is written object{property:value;}
  • A margin indicates how much space we want around the outside of an element
  • A padding indicates how much space we want around the content inside an element
  • you can put multiple css properties inside the brackets for one object
  • css goes in the header
  • you can put multiple objects seperated by a comma to augment with the css properties intide the brackets following

Git

  • notes
  • git status: checks what branch you are in
  • git checkout -b branch-name: creates a new branch
  • code .: opens file in vs code and start: opens a regular file
  • cd file-name: moves into a file and cd..: moves back 1 sstep out of a file
  • mk dir folder-name: makes a new folder and touch file-name.end: creates a file
  • pwd: print working directory shows what file you are currently in
  • ls: lists files in current folder
  • git add -A: adds all changes to branch
  • Git checkout branch-name: go to branch
  • Git push origin branch-name: Pushes code from branch to main
  • Git pull origin main: Makes sure you are working on current version of code
  • Git commit -m “commit comment here”:
  • Git config --global user.name “name”: set user name or Git config --global user.email “email@email.com”: set email
